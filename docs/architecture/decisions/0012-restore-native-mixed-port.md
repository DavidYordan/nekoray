# ADR 0012：恢复 NekoRay 原生 Mixed 默认端口

状态：Accepted
日期：2026-07-24

## 背景

ADR 0006 根据当时本机另一套生产 NekoRay 占用 `2080` 的环境，将本项目主 Mixed 默认端口迁移为 `12080`。该外部生产 NekoRay 已不再作为本机基础网络；本机现使用 Clash TUN。用户明确要求本项目恢复 NekoRay 原生默认值。

辅助 Mixed 是本项目新增的并发线路入口，不跟随主端口回迁；已分配的 `12100`、`12101` 及默认池 `12100..12299` 保持不变。

## 决策

- 新配置的主 Mixed 默认监听 `127.0.0.1:2080`。
- 当前私人部署中仍为旧项目默认值 `12080` 的主端口，显式迁移到 `2080`。
- 不自动改写用户已经选择的其它自定义主端口；代码默认值变化只作用于没有持久值的新配置。
- 辅助端口、profile 到辅助端口的绑定及其严格路由语义不变。
- 诊断必须核对端口监听 PID 与可执行文件路径；“端口可连”不能单独证明它属于本项目或目标线路可用。
- Clash TUN 是本机外部底层网络，不得把其接口名、Fake-IP 网段、物理网卡或 `auto_detect_interface` 诊断覆盖硬编码进产品配置。

## 迁移与回滚

当前部署配置修改前必须生成可恢复副本。若 `2080` 已被其它进程占用，启动应明确失败，不得回退到 `12080` 或随机端口。回滚只能由用户明确把主端口改回其选择值；程序不得静默双监听或端口转发。

## 影响

- 主入口契约更新为 `2080 -> 当前主 profile 完整 chain`。
- 辅助入口继续保持“哪个端口进入，就只从该端口绑定的线路出去”。
- ADR 0006 保留为历史环境决策，不再代表当前默认值。

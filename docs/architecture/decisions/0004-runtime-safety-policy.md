# ADR 0004：Windows 运行时与网络模式安全不变量

状态：Accepted
日期：2026-07-20
最后复核：2026-07-22

## 背景

系统代理和 TUN 是 Windows OS 状态，不等同于 GUI 中保存的布尔意图。当前 GUI、sing-box worker 和内部 TUN 生命周期耦合；只在退出时保留注册表或状态标记，会形成指向已停止入口的黑洞，而整核重启内部 TUN 还可能产生默认直连窗口。

本 ADR 冻结产品必须满足的行为，不选择具体服务、WFP 或热切换实现。候选实现见 [ADR 0008](0008-persistent-windows-runtime.md)。

## 决策

- 只有用户发起的、目标精确且结果可核对的手动操作，才允许启用或停用系统代理、启用或停用 TUN。
- GUI 退出、GUI/core 重启、线路切换、路由重载、辅助端口变化和更新流程不得自行改变系统代理或 TUN 的 OS 模式。
- 启动时可以观察并展示 OS 状态，但不得因为持久化的 `enabled`、重启参数或“记住上次状态”而自动调用启停操作。
- 当系统代理或 TUN 处于启用状态时，GUI 生命周期不得停止它所依赖的数据面。GUI 退出应只断开控制面；后续运行时必须继续持有入口、路由和保护状态。
- TUN 开启时必须能够完成线路重启或切换，不能要求用户先关闭 TUN。切换期间绝不允许 IPv4、IPv6 或 DNS 直连偷跑。
- core 不健康或候选线路启动失败时，允许进入明确可观察的全阻断状态，但不得退回无保护直连。提交后故障不得自动切回旧线路；只有用户明确选择且旧 generation 重新验证后才允许显式回滚。黑洞不等于“顺利重启”，可用性必须单独验收。
- UI 的 Mixed、系统代理和 TUN 状态必须来自实际监听、精确进程身份和 Windows 状态；持久化意图不能作为成功证据。
- 任何停止、恢复或清理操作只允许作用于本实例精确拥有的 PID、句柄、创建时间、路径、注册表值、接口、路由和过滤器。
- 只有用户手动关闭系统代理时，才允许 compare-and-restore；并且只有当前 PAC、手工代理和 bypass 仍与本项目写入值匹配时，才可恢复此前快照。
- `D:\Program Files\nekoray`、其 `2080` 入口以及它持有的生产 TUN 均是本项目之外的受保护状态，不得被本项目停止、恢复或接管。

## 验收含义

“未观察到直连”必须由 Windows API、路由、DNS、IPv4/IPv6 流量和进程所有权共同取证。单纯保留系统代理注册表、保留一个 TUN 意图布尔值、让请求失败或要求用户先关闭 TUN，均不构成此 ADR 的验收。

当前实现尚未满足本 ADR。普通启动/重启的自动模式恢复与无令牌 TUN 提权连续流程已经移除；不精准的 Windows 系统代理 UI 写入也已临时禁用。Windows GUI 还会忽略 CRT `SIGTERM`/`SIGINT`，防止这两个控制台信号绕过 UI exit guard；它不覆盖强杀、崩溃、系统关机或 worker 自退。GUI/core 生命周期耦合、内部 TUN 整核重载、外置 TUN 身份交接、精准系统代理 broker 和独立 kill-switch 仍需整改。

普通 GUI 的 localhost 随机令牌 gRPC 与用户显式执行的 `nekobox_core run/check` 必须分开建模。前者不是任意无认证入口；后者是维护/测试高级入口，可直接读取配置。当前 Go core 尚未重复 C++ 产品策略校验，属于需要在 Runtime/core 边界补齐的纵深防御，不能用 `check` 成功代替本 ADR 验收。

## 不在本 ADR 中的决定

- 线路切换是否允许短暂全阻断和既有连接重置。
- Windows 重启后如何在不自动启用 TUN 的前提下维持保护、展示状态并等待用户手动操作。
- kill-switch 是全机还是按进程保护，以及 LAN、DHCP、打印机和 RDP 例外。

这些实现和体验细节保留在 ADR 0008 与待决策清单中，但不得削弱本 ADR 的“仅手动改变 OS 模式”和“绝不直连”约束。

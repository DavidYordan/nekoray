# ADR 0006：Mixed 默认端口与生产实例隔离

状态：Accepted
日期：2026-07-20

## 背景

本机 `D:\Program Files\nekoray` 中运行着另一套生产 Nekoray，其 Mixed 入口使用 `127.0.0.1:2080`，并持有本机必须保持的生产 TUN。它不是本项目的旧部署、兼容目标或可由测试接管的资源。

本项目当前默认主 Mixed 端口为 `12080`。用户已经确认继续使用该默认值。

## 决策

- 新配置和默认配置的主 Mixed 端口使用 `12080`。
- 本项目不得监听、迁移、复用或自动接管 `2080`，也不得停止占用它的生产实例。
- 不提供从生产实例 `2080` 到本项目 `12080` 的自动迁移、兼容监听或端口转发。
- 用户显式保存的其他合法端口是用户配置，不得仅因默认值变化而静默重写。
- UI 和诊断必须区分“配置端口”“实际监听 PID/路径”和“本项目是否拥有该监听”。
- `2080` 请求成功只能证明外部生产实例可用，不能作为本项目 Mixed 或 outbound 的验收证据。

## 影响

- 本项目测试和文档示例默认使用 `12080` 或经过空闲检查的临时端口。
- 所有按名称批量停止 `nekobox_core.exe` 或 `nekoray.exe` 的行为均违反本决策；进程操作必须验证精确路径和身份。
- 构建、打包和测试不得把 `D:\Program Files\nekoray` 当成可修改的部署目录。
- Mixed 的端口到线路映射由 [ADR 0007](0007-mixed-routing-contract.md) 定义，与默认端口选择分开。
